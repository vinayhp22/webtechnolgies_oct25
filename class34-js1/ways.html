Ways to Add JS to Webpages

1. Inline JS

Advantages of Inline JS

Simple & quick for small functionality

Easy to test short code snippets.

Disadvantages of Inline JS

Not maintainable because its mixing both HTML & JS

Difficult to reuse the same logic for multiple elements

Considered bad practice in modern web development.

2. Internal JS

Just reload!!!, you'll get the welcome message

Advantages of Internal JS

Keeps JS separate from HTML elements

Better than Inline JS for larger scripts

Useful when we only have one HTML file/page.

Disadvantages of Internal JS

if the HTML grows big, having CSS, and JS all in the same file gets cluttered(tough maintainable)

Code is not reusable across multiple pages.

3. External JS - i.e., separate .js file

Advantages of External JS

Best Practice in modern web development

Clean separation between HTML, CSS and JS(Structure, Style, behaviour)

Easier to maintain and debug

Can be reused across multiple web pages by linking the same .js file

Improves page loading speed(browser can cache that .js file)

Disadvantages of External JS

Requires multiple files..

If the External .js file is large, then again it may slow down first page load itselt.(but this can be fixed using async/defer)
